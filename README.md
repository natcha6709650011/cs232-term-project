ระบบ TU Class เป็นระบบเช็คชื่อเข้าเรียนผ่าน LINE Official Account (LINE OA) โดยใช้ LINE Front-end Framework (LIFF) เป็นตัวกลางในการใช้งาน และใช้ Backend บน AWS ในการประมวลผลข้อมูล

ระบบนี้ถูกออกแบบมาเพื่อ:
 • ยืนยันตัวตนผู้ใช้งานผ่านระบบของมหาวิทยาลัย
 • ป้องกันการโกงการเช็คชื่อ
 • รองรับการเรียนทั้งแบบ Onsite และ Online
 • สรุปผลการเข้าเรียนโดยอัตโนมัติ

⸻

2. ภาพรวมการทำงานของระบบ (System Overview)

ผู้ใช้งานจะเข้าสู่ระบบผ่าน LINE OA และทำการ login ผ่าน LIFF โดยระบบจะทำการตรวจสอบตัวตนผ่าน TU API จากนั้นจะแยกบทบาทเป็น อาจารย์ (Teacher) และ นักศึกษา (Student)

ระบบประกอบด้วยฟีเจอร์หลักดังนี้:

ฝั่งอาจารย์
 1. เริ่มคาบเรียน (Onsite)
 2. เรียนออนไลน์ / ยกคลาส
 3. เช็คสถานะ
 4. ติดต่อเจ้าหน้าที่

ฝั่งนักศึกษา
 1. เช็คชื่อ
 2. ลา
 3. ดูสถานะ
 4. ติดต่อเจ้าหน้าที่

⸻

3. กระบวนการยืนยันตัวตน (Authentication Process)

ผู้ใช้งานเข้าสู่ระบบผ่าน LINE OA
→ เปิด LIFF Application
→ ระบบดึง line_user_id

ผู้ใช้กรอก:
 • Username / Email
 • Password

Backend ทำการเรียก TU Authentication API เพื่อตรวจสอบข้อมูล

กรณีตรวจสอบสำเร็จ
 • ผูก line_user_id กับบัญชีผู้ใช้
 • บันทึกลงฐานข้อมูล
 • กำหนดบทบาทผู้ใช้งาน
 • เข้าสู่หน้า Home

กรณีตรวจสอบไม่สำเร็จ
 • แสดงข้อความผิดพลาด
 • ไม่อนุญาตเข้าใช้งาน

⸻

4. การทำงานฝั่งอาจารย์ (Teacher Functions)

4.1 ฟีเจอร์: เริ่มคาบเรียน (Onsite)

อาจารย์กดเริ่มคาบเรียน
→ ระบบร้องขอสิทธิ์เข้าถึงตำแหน่ง (GPS)
→ ดึงค่า latitude และ longitude ของอาจารย์
→ บันทึกตำแหน่งลงใน session

Backend สร้าง session โดยมีข้อมูล:
 • session_id
 • teacher_id
 • ประเภท session = onsite
 • teacher_location (latitude, longitude)
 • เวลาเริ่มต้น
 • เวลาหมดอายุ (30 นาที)

จากนั้น:
→ สร้าง check-in link หรือ QR Code
→ ส่งผ่าน LINE Bot ไปยังนักศึกษา

⸻

4.2 ฟีเจอร์: เรียนออนไลน์ / ยกคลาส

อาจารย์เลือกประเภทการเรียน

กรณีเรียนออนไลน์ (Online)
 • ไม่ใช้ตำแหน่ง
 • Backend สร้าง session
 • กำหนดอายุ 30 นาที
 • ส่ง link/QR ให้นักศึกษา

กรณียกคลาส (Cancel)
 • บันทึกสถานะเป็นยกเลิก
 • แจ้งนักศึกษาผ่าน LINE Bot
 • ไม่มีการเช็คชื่อ

⸻

4.3 ฟีเจอร์: เช็คสถานะ

อาจารย์สามารถดูสถานะการเข้าเรียนของนักศึกษา

Backend ดึงข้อมูลจากฐานข้อมูลและแสดง:
 • จำนวนผู้เข้าเรียน (Present)
 • จำนวนผู้ขาดเรียน (Absent)
 • จำนวนผู้ลา (Leave)
 • รายชื่อพร้อมสถานะรายบุคคล

⸻

4.4 ฟีเจอร์: ติดต่อเจ้าหน้าที่

ผู้ใช้งานสามารถติดต่อเจ้าหน้าที่ผ่าน LINE Support Channel เพื่อแก้ไขปัญหาการใช้งาน

⸻

5. การทำงานฝั่งนักศึกษา (Student Functions)

5.1 ฟีเจอร์: เช็คชื่อ (Check-in)

นักศึกษาได้รับ link หรือ QR Code จาก LINE Bot
→ เปิดผ่าน LIFF

ระบบดำเนินการ:
 • เปิดกล้อง (Camera Only)
 • นักศึกษาถ่ายภาพแบบเรียลไทม์
 • ดึงตำแหน่ง GPS

จากนั้นส่งข้อมูลไปยัง Backend:
 • session_id
 • student_id
 • line_user_id
 • latitude, longitude
 • image_url

⸻

5.2 การตรวจสอบของ Backend (Validation)

Backend ตรวจสอบเงื่อนไขดังนี้:
 1. ผู้ใช้งานตรงกับบัญชี (line_user_id)
 2. session ยังไม่หมดอายุ
 3. มีภาพยืนยันตัวตน

กรณี Onsite
 • ดึงตำแหน่งของอาจารย์จาก session
 • คำนวณระยะระหว่างอาจารย์และนักศึกษา
 • นักศึกษาต้องอยู่ในระยะไม่เกิน 50 เมตร

หากเกินระยะ:
→ ไม่อนุญาตให้เช็คชื่อ

⸻

5.3 ผลลัพธ์การเช็คชื่อ
 • ผ่านเงื่อนไข → บันทึกสถานะเป็น Present
 • ไม่ผ่าน → แสดงข้อความแจ้งเตือน

⸻

5.4 ฟีเจอร์: ลา (Leave)

นักศึกษากรอก:
 • เหตุผลการลา
 • แนบเอกสาร (ถ้ามี)

Backend บันทึกสถานะเป็น Leave

⸻

5.5 ฟีเจอร์: ดูสถานะ

นักศึกษาสามารถดูประวัติการเข้าเรียนของตนเอง:
 • Present
 • Absent
 • Leave

⸻

5.6 ฟีเจอร์: ติดต่อเจ้าหน้าที่

เปิดช่องทางติดต่อผ่าน LINE Support

⸻

6. การสรุปผลอัตโนมัติ (Automatic Attendance Processing)

เมื่อ session หมดอายุ (30 นาที):

Backend ทำการสรุปผลอัตโนมัติ:
 • มีการ check-in → Present
 • มีการลา → Leave
 • ไม่มีข้อมูล → Absent

ผลลัพธ์ถูก:
 • บันทึกลงฐานข้อมูล
 • ส่งรายงานให้อาจารย์ผ่าน LINE หรือ Email

⸻

7. หลักการสำคัญของระบบ (Key System Logic)

7.1 การตรวจสอบตำแหน่ง (Location Validation)
 • ใช้เฉพาะการเรียนแบบ Onsite
 • ใช้ตำแหน่งอาจารย์เป็นจุดอ้างอิง
 • นักศึกษาต้องอยู่ในรัศมีไม่เกิน 50 เมตร

⸻

7.2 การควบคุม Session
 • ทุก session มีอายุ 30 นาที
 • ไม่สามารถเช็คชื่อหลังหมดเวลา

⸻

7.3 การป้องกันการโกง (Anti-Cheating)
 • ใช้ LINE ID ผูกกับบัญชีผู้ใช้เพียง1บัญชี
 • ต้องถ่ายภาพจากกล้องเท่านั้น
 • ห้ามอัปโหลดรูปจากเครื่อง
 • ตรวจสอบตำแหน่งแบบเรียลไทม์

⸻

7.4 ประเภทสถานะการเข้าเรียน
 • Present
 • Absent
 • Leave

⸻

8. สรุป

ระบบ TU Class Attendance System เป็นระบบเช็คชื่อที่ใช้ LINE เป็นแพลตฟอร์มหลัก โดยมีการตรวจสอบตัวตนผ่านระบบของมหาวิทยาลัย ร่วมกับการยืนยันตัวตนผ่านภาพถ่ายและตำแหน่ง GPS เพื่อเพิ่มความถูกต้องและลดการทุจริตในการเช็คชื่อ อีกทั้งยังมีระบบสรุปผลอัตโนมัติที่ช่วยลดภาระของอาจารย์ในการจัดการข้อมูลการเข้าเรียน
